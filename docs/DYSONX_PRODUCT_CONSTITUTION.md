# DysonX Product Constitution v1.0

DysonX is not a generic news aggregation website, RSS mirror, AI summary blog, or content farm.

DysonX is an **AI / AGI Intelligence OS**. It tracks first-source signals shaping AI, AGI, foundation models, world models, agents, robotics, multimodal AI, AI infrastructure, AI safety, AI policy, open source AI, compute, and research-to-business translation.

## Supreme Rule

Every product, engineering, content, SEO, automation, and distribution decision must be judged by one question:

**Does this increase long-term intelligence value?**

If the answer is no, the feature should not be built.

## Core Value

News is the input. Insight is the product. The knowledge graph is the asset.

DysonX competes on:

1. First-source monitoring
2. Source authority and credibility scoring
3. AGI impact analysis
4. Entity and relationship extraction
5. Long-term company, person, topic, model, paper, and policy tracking
6. Prediction and later review
7. Decision usefulness
8. Reusable intelligence assets

## Language Rule

English is the default language of the product. Chinese is a user-switchable localization layer.

- Root or `/en` defaults to English.
- `/zh` provides Chinese localization.
- Canonical routes, metadata, structured data, Open Graph, X/Twitter cards, and sitemap entries default to English.
- Chinese fields must be parallel localized fields, not canonical identifiers.
- If only one language is available, English wins.

## Primary Content Object: Signal

The primary content object is **Signal**, not Article.

Every Signal must answer:

- What happened?
- What is the original source?
- How authoritative is it?
- Why does it matter?
- Which AGI capability does it affect?
- Which companies, people, products, models, papers, projects, or policies are involved?
- What is DysonX’s interpretation?
- What should be watched next?

Required Signal fields:

- Signal ID
- Title
- Slug
- Summary
- Original source URL
- Source type
- Source authority score
- Confidence score
- AGI impact score
- Market impact score
- Technical impact score
- Affected AGI capabilities
- Related entities
- DysonX Take
- Watch Next
- Duplicate group ID
- Publish status
- English version
- Chinese version, when available
- SEO metadata
- Social distribution drafts
- Created time
- Updated time

## First-Source Rule

Prioritize first-source information:

- Official company blogs
- Official research labs
- Official product announcements
- Research papers
- GitHub repositories and releases
- Government and regulatory sources
- Direct statements from key people
- Official conference, keynote, and technical materials
- Official standards bodies

No Signal should be published without source attribution.

## Notion Source Rule

Monitoring sources must not be permanently hardcoded. All monitored sources must be dynamically managed through a Notion database.

Code may cache source configuration for performance, but Notion remains the source of truth.

## LLM-First Analysis Rule

After raw content is collected, the next major processing step must be LLM analysis.

The system must not rely only on fixed keyword rules for classification, importance, AGI relevance, or interpretation.

Canonical pipeline:

`Source -> LLM Analysis -> Structured Knowledge -> Publish / Distribute`

Forbidden pipeline:

`Source -> Rule Engine -> Generic Article`

## Knowledge Graph Rule

DysonX must not store content only as flat pages.

Core entities include Signal, Source, Company, Person, Organization, Government body, Product, Model, Paper, GitHub project, Policy, Technology topic, AGI capability, Event, Prediction, and Report.

The website is only the presentation layer. The knowledge graph is the long-term asset.

## Trackers and Reports

Trackers are persistent intelligence surfaces for companies, people, models, topics, capabilities, papers, policies, and GitHub projects.

Reports synthesize multiple Signals and must not merely concatenate articles.

## Anti-Garbage Rule

Reject content that is only:

- A generic AI summary
- A rewritten third-party news article
- A list of facts without interpretation
- A rumor without labeling
- A duplicate of an existing Signal
- A low-relevance AI trend post
- A generic SEO article

If content does not improve the knowledge graph, tracker pages, AGI Map, reports, SEO authority, social distribution value, or user decision-making, it should not be published.

## Engineering Priority Order

When tradeoffs occur, prioritize:

1. Long-term intelligence value
2. Source authority
3. Structured knowledge graph
4. AGI impact interpretation
5. User decision usefulness
6. Data integrity
7. SEO discoverability
8. Publishing speed
9. Visual polish
10. Content volume

Speed must never override credibility. Volume must never override value.

## Codex Mandate

Codex is not building a news website.

Codex is building an AI / AGI intelligence system where news is only the raw material, Signals are the operating unit, the knowledge graph is the asset, and decision value is the product.
